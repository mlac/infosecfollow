1. Oracle PeopleSoft zero-day exploited by ShinyHunters

[zero-day, breach, patch]

Last 24h: Researchers and CISA confirmed active exploitation of CVE-2026-35273 in Oracle PeopleSoft/PeopleTools, with Google attributing data theft from hundreds of organizations to ShinyHunters.

CVE-2026-35273 is a missing-authentication-for-critical-function flaw in Oracle PeopleSoft Enterprise PeopleTools that allows unauthenticated attackers to steal large volumes of data. The ShinyHunters group has weaponized it broadly, disproportionately affecting American universities running the ERP software. Oracle has issued mitigations but has not publicly confirmed in-the-wild exploitation; CISA added the bug to its Known Exploited Vulnerabilities catalog. Organizations running PeopleSoft should apply Oracle's fixes immediately and hunt for signs of bulk data exfiltration.

2. Ivanti Sentry flaw under attack, CISA issues 3-day patch order

[zero-day, patch, policy]

Last 24h: CISA invoked new Binding Operational Directive 26-04 to give federal agencies three days to patch an actively exploited Ivanti Sentry flaw, as exploitation attempts hit honeypots.

A critical OS command injection vulnerability in Ivanti Sentry allows attackers to execute arbitrary code with root privileges. Security researchers are already observing exploitation attempts against internet-facing honeypots, indicating opportunistic scanning and attacks. CISA ordered federal agencies to remediate by Sunday under the newly issued BOD 26-04, which prioritizes patching based on risk. Administrators should patch Sentry immediately and review systems for signs of root-level compromise.

3. Over 400 Arch Linux AUR packages hijacked

[supply-chain, malware, rootkit]

Last 24h: Attackers took over more than 400 packages in the Arch User Repository, rewriting build scripts to deploy a Rust infostealer and an eBPF rootkit.

The Arch User Repository (AUR) is Arch Linux's community package collection where build scripts run on users' machines. Attackers compromised over 400 packages and altered their build scripts to install a Rust-based credential stealer that harvests developer secrets and access tokens. When it runs with root, the malware can also load an eBPF rootkit to hide its presence. Arch users who built any AUR packages this week should assume credential compromise, rotate secrets, and inspect systems for the rootkit.

Sources: The Hacker News · BleepingComputer

[ai, phishing, zero-day]

Teams deploying AI agents should patch LangGraph, scrutinize agent inputs, and govern agent behavior using NIST and ISO frameworks.

7. Velvet Ant backdoored Linux login software for nearly a decade

[apt, backdoor]

Last 24h: Sygnia disclosed that China-nexus group Velvet Ant hid inside Linux PAM and OpenSSH login components for close to ten years to evade detection.

Rather than hiding on closely monitored endpoints, the China-linked Velvet Ant group backdoored the PAM and OpenSSH components that govern authentication on a targeted network. By embedding access in the Linux login system itself, the attackers placed their foothold where routine cleanup and standard monitoring could not reach. The intrusion persisted for nearly a decade. Defenders should verify the integrity of PAM modules and SSH binaries and treat authentication infrastructure as a high-value detection target.
